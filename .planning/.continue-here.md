<decisions_made>

- Architecture: Dispatcher + K8s Job per scan (Approach C from design)
- Same Docker image for both dispatcher and scan runner, different entrypoints
- Scanner binary baked into image at build time + `scanrook upgrade` at container startup
- Resource tiers: Small (<500MB, 1/2 CPU, 1/3Gi), Medium (500MB-5GB, 2/4 CPU, 2/6Gi), Large (>5GB, 3/6 CPU, 4/10Gi)
- Max concurrent: Small=6, Medium=3, Large=1
- Dispatcher checks S3 object size via HeadObject before creating Job
- Job TTL: 300s after completion
- No K8s-level retries (backoffLimit=0), dispatcher handles retry logic
- Progress streaming unchanged: scan runner writes to DB directly, SSE pipeline identical
</decisions_made>
